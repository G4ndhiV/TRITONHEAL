\section{Comparación crítica (2.4)}

\textbf{Frontera Ollama (Coder-V2)} lidera MSR con latencia mediana baja ($\sim$2\,s) y JSON 100\%. \textbf{Llama-8B} es el SLM local operativamente viable. \textbf{R1-Distill-14B} no es comparable en igualdad de condiciones sin ajustar timeout o cuantización. \textbf{Dual} equilibra detección (MSR 80.6\%) y F1 (0.86) frente a veto excesivo. \textbf{APIs heurísticas} igualan MSR entre sí pero no reflejan latencia ni razonamiento real de frontera comercial.

\section{Amenazas a la validez observadas (2.5)}

\textbf{Validez interna.} ¿Se controlaron semillas y versiones? Sí para benchmark (seed 42) y tags Ollama fijos; no para APIs (heurístico). ¿Hardware uniforme? Un solo M4 24\,GB. ¿Timeouts? R1-14B con 14.3\% JSON válido sesga MSR hacia cero.

\textbf{Validez externa.} ¿Generaliza a otros GPUs? No probado (Metal vs.\ CUDA). ¿Otros dominios? Benchmark de seguridad Triton, no cargas de producción.

\textbf{Validez de constructo.} ¿MSR engaña? Sí si se confunde con seguridad en ejecución GPU. ¿JSON válido engaña? Sí como ``compilación''; solo indica parseo del verificador. ¿Heurísticos de API? Sí: inflan acuerdo con ground truth sin costo de inferencia real.

\textbf{Mitigaciones aplicadas.} Reclasificación DeepSeek, reporte de JSON válido y timeouts, mapeo rúbrica (1.5), matriz de cumplimiento (apéndice).

\section{Discusión (2.6)}

\textbf{¿Evidencia para hipótesis?} Descriptivamente PRI-1 favorece Coder-V2; PRI-2 favorece dual en FNR; inferencia confirmatoria individual es conservadora tras Holm. \textbf{Limitaciones:} sin generación ni speedup medidos. \textbf{Qué haríamos distinto:} APIs reales, segundo hardware NVIDIA, timeout adaptativo para R1-14B, corrida de generación+compilación por configuración. \textbf{Amenazas abiertas:} constructo MSR vs.\ GPU; heurísticos. \textbf{Escala:} Coder-V2 en Ollama compite con SLM 8B en este host.

\section{Conclusión (2.7)}

\begin{enumerate}[noitemsep]
  \item \textbf{Objetivo:} Se evaluó el pipeline generativo-verificativo vía verificador SLM; proxies alineados a rúbrica con limitaciones documentadas.
  \item \textbf{$H_0$ confirmatorios:} No rechazados en C1--C3 y C5 tras Holm; Friedman global sí rechaza igualdad de F1.
  \item \textbf{Efecto práctico:} Coder-V2 $\approx$87\% MSR; dual $\approx$81\% con F1 0.86; Llama local $\approx$65\%.
  \item \textbf{Reproducibilidad:} Repo \texttt{G4ndhiV/TRITONHEAL}, commit \texttt{2c027b5}, \texttt{make stats figures pdf}.
  \item \textbf{Métricas engañosas:} JSON válido $\neq$ compilación de kernel generado; MSR $\neq$ seguridad en GPU.
  \item \textbf{Debilidad principal:} R1-14B local inoperante en esta corrida y APIs no reales.
  \item \textbf{Taxonomía:} Coder-V2 = frontera; R1-Distill-14B = SLM local.
\end{enumerate}
